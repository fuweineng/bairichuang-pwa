## 二、音频播放修复

### 问题
- 英语听力题部分 audioUrl 指向本地文件（相对路径），PWA 离线时无法播放
- english-201 完全缺音频文件
- 部分 m4a 格式在 iOS Safari 兼容性差

### 修复方案
1. `playAudio()` 函数：优先用 `speechSynthesis` TTS 读 `audio_text`；无 `audio_text` 时 fallback 到 `audioUrl`
2. `english-201` 缺音频：使用 TTS 生成音频（`say` → ffmpeg → m4a）
3. 离线检测：若无网络且无本地缓存，显示"音频暂无，请检查网络"

## 三、题库扩容

### 目标
每个考点（knowledgeTag）至少 5 题（不同变化题型）。

### 优先级
1. **英语**：113 个考点不足 5 题，共约 500 题
2. **物理**：58 个考点
3. **化学**：80 个考点
4. **历史/地理/道法**：各 40+ 考点
5. **数学**：38 个考点
6. **生物/语文**：各 30 个考点

### 题型规则
- **choice**：4 选项（标准中考格式）
- **fill**：填空（1-3 空）
- **listening**：音频 + 3-4 选项
- **dictation**：听写（音频读句子，学生写）
- **short_answer**：简答（2-3 分答案）

## 四、英语音频生成

### 文件路径
`audio/english/english-NNN.m4a`

### 生成流程
```bash
# 1. 用 macOS say 生成 aiff
say -v Ting-Ting -o english-XXX.aiff "text here"

# 2. ffmpeg 转 m4a
ffmpeg -i english-XXX.aiff -acodec alac english-XXX.m4a
```

### 覆盖范围
- 现有 180 个有 audioUrl 的题目：保持原样
- english-201：需要生成新音频（当前缺）
- 30 个 listening 类型（english-206 ~ 235）：检查文件存在性，缺失则生成
